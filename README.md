## Цель работы

Изучить методы построения внутреннего представления программы на основе контекстно-свободной грамматики, реализовать синтаксический анализатор методом рекурсивного спуска и преобразовать арифметические выражения в тетрады и ПОЛИЗ.

## Постановка задачи

Реализовать программу, которая выполняет:

- лексический анализ арифметического выражения;
- синтаксический анализ методом рекурсивного спуска;
- поиск лексических и синтаксических ошибок;
- построение внутреннего представления программы в виде тетрад;
- построение ПОЛИЗ;
- вычисление арифметического выражения, если оно состоит только из целых чисел.

## Контекстно-свободная грамматика

```text
E → T A
A → ε | + T A | - T A

T → F B
B → ε | * F B | / F B | // F B | % F B | ** F B

F → num | id | ( E )

id  → letter { letter | digit | _ }
num → digit { digit }
```

## Терминальные символы

```text
+  -  *  /  //  %  **  (  )
```

| Тип лексемы | Описание |

| `NUMBER` | Целое число |

| `LPAREN` | Открывающая скобка |

## Лексический анализатор

Лексический анализатор выполняет разбиение входной строки на лексемы.

На этапе лексического анализа выделяются:

- идентификаторы;
- целые числа;
- арифметические операторы;
- круглые скобки;
- недопустимые символы.

## Лексические и синтаксические ошибки

Программа обрабатывает следующие ошибки:

- пустая строка;
- недопустимый символ;
- пропущенный операнд;
- лишний оператор;
- лишняя закрывающая скобка;
- пропущенная закрывающая скобка.

При наличии ошибок:

- тетрады не строятся;
- ПОЛИЗ не формируется;
- вычисление не выполняется.

## Примеры лексических ошибок

### Недопустимый символ

**Входная строка:**

```text
5 + @
```

**Ожидаемый результат:**

```text
Лексическая ошибка: Недопустимый символ '@'
```

### Недопустимый символ

**Входная строка:**

```text
3 $ 4
```

**Ожидаемый результат:**

```text
Лексическая ошибка: Недопустимый символ '$'
```

## Примеры синтаксических ошибок

### Пропущен операнд

**Входная строка:**

```text
3 + * 4
```

**Ожидаемый результат:**

```text
Синтаксическая ошибка: Неожиданный токен '*'. Ожидался операнд.
```

### Пропущена закрывающая скобка

**Входная строка:**

```text
(5 + 3
```

**Ожидаемый результат:**

```text
Синтаксическая ошибка: Ожидалась 'RPAREN'
```

### Лишняя закрывающая скобка

**Входная строка:**

```text
5 + 3)
```

**Ожидаемый результат:**

```text
Синтаксическая ошибка: Неожиданный токен ')' после завершения выражения.
```

## Внутренняя форма представления программы

Внутренняя форма представления программы реализована в виде тетрад.

Формат тетрады:

```text
(op, arg1, arg2, result)
```

## Описание полей тетрады

| Поле | Описание |
|---|---|
| `op` | Оператор |
| `arg1` | Первый аргумент |
| `arg2` | Второй аргумент |
| `result` | Временная переменная |

Временные переменные создаются автоматически:

```text
t1, t2, t3, ...
```

## Пример построения тетрад

**Входная строка:**

```text
3 + 5
```

**Тетрады:**

| № | Оператор | Аргумент 1 | Аргумент 2 | Результат |
|---|---|---|---|---|
| 1 | `+` | `3` | `5` | `t1` |

## Пример построения тетрад с приоритетом операций

**Входная строка:**

```text
2 + 3 * 4
```

**Тетрады:**

| № | Оператор | Аргумент 1 | Аргумент 2 | Результат |
|---|---|---|---|---|
| 1 | `*` | `3` | `4` | `t1` |
| 2 | `+` | `2` | `t1` | `t2` |

## ПОЛИЗ

ПОЛИЗ — польская инверсная запись.

ПОЛИЗ формируется только для выражений, состоящих исключительно из целых чисел.

Если в выражении присутствуют идентификаторы:

- тетрады строятся;
- ПОЛИЗ не строится;
- вычисление не выполняется.

## Пример построения ПОЛИЗ

**Входная строка:**

```text
3 + 5
```

**ПОЛИЗ:**

```text
3 5 +
```

**Результат вычисления:**

```text
8
```

## Пример вычисления выражения с приоритетом операций

**Входная строка:**

```text
2 + 3 * 4
```

**ПОЛИЗ:**

```text
2 3 4 * +
```

**Результат вычисления:**

```text
14
```

## Пример вычисления выражения со скобками

**Входная строка:**

```text
(2 + 3) * 4
```

**ПОЛИЗ:**

```text
2 3 + 4 *
```

**Результат вычисления:**

```text
20
```
